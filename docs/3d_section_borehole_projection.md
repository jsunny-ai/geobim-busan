# 3D 수직단면 시추공 투영 방식

- 적용 대상: `sites/viewer-3d`
- 작성일: 2026-07-14
- 상태: 구현 기준 문서

## 1. 목적

수직단면에서 원래 3D 위치의 시추공 원통과 카메라를 향하는 Sprite 라벨을 그대로 표시하면, 단면에서 멀리 떨어진 시추공도 화면상 단면 위에 겹치거나 카메라 회전 시 단면 앞뒤로 돌출되어 보인다.

수직단면 모드에서는 절단 평면이 시추공 원통을 실제로 통과하는 경우에만 단면 전용 기둥을 표시한다. 절단면 뒤쪽의 유지 반공간에 있는 다른 시추공은 위치 확인용 시추공명만 남기며, 원본 기둥·collar 마커와 제거 반공간의 표식은 숨긴다. 지층 솔리드는 기존 절단 평면과 절단 캡을 계속 사용한다.

## 2. Leapfrog 동작에서 채택한 원칙

Seequent Leapfrog의 단면 레이아웃은 시추공을 단면에 투영하고, 시추공 궤적에서 단면까지의 최소거리로 포함 여부를 필터링한다. 필요하면 허용거리 밖의 궤적을 자를 수 있다. 일반 3D 슬라이서는 앞/뒤 제거와 일정 폭의 Thick Slice를 별도로 제공한다.

- [Organising Models and Other Objects on the Section](https://help.seequent.com/Geo/2026.1/en-GB/Content/presentation/layout-models.htm)
- [Section Layouts](https://help.seequent.com/Geo/2026.1/en-GB/Content/presentation/section-editor.htm)
- [Slicing Through Data in the Scene](https://help.seequent.com/Geo/2025.3/en-GB/Content/basics/slicer.htm)

우리 뷰어는 직선 수직단면만 지원하므로 엄밀한 직교 투영을 사용한다. 곡선 장단면용 illustrative projection은 적용하지 않는다.

## 3. 기존 구현의 문제

1. 하나의 절단 평면은 반공간 한쪽만 제거하므로, 유지되는 쪽의 모든 시추공이 남는다.
2. 시추공 그룹의 모든 자식을 `visible=true`로 강제해 단면 거리 필터가 없다.
3. 원통과 Sprite 라벨은 실제 3D 위치에 있으므로 정면에서는 단면 위에 겹쳐 보이고, 회전하면 앞뒤로 돌출된다.
4. Sprite 라벨은 항상 카메라를 향하므로 단면에 고정된 도면 요소처럼 보이지 않는다.

## 4. 렌더링 계층

```text
일반 3D 모드
└─ Borehole3DGroup
   ├─ 실제 위치의 원통형 지층 기둥
   ├─ collar 마커
   └─ billboard 라벨

수직단면 모드
├─ Borehole3DGroup
│  ├─ 남는 반공간: 단면 비관통 시추공명만 유지
│  ├─ 원본 3D 기둥·외곽선·collar 마커: 숨김
│  └─ 제거되는 반공간의 표식: 숨김
├─ 기존 지층 솔리드: 절단 평면 + 절단 캡 유지
└─ SectionBoreholeGroup
   ├─ 단면에 직교 투영된 지층별 평면 리본
   ├─ 리본 윤곽선
   └─ 단면 평면과 나란한 시추공명 라벨
```

뒤쪽 시추공명은 시추공의 평면 위치 탐색용이고, `SectionBoreholeGroup`은 절단 평면이 실제 통과한 시추공의 지질 확인용이다. 단면 종료 시 `SectionBoreholeGroup`을 폐기하고 모든 원본 시추공 형상의 표시 상태를 복원한다.

## 5. 투영 계산

단면 평면을 `n · p + c = 0`이라고 할 때 시추공 점 `p`의 부호 있는 거리는 다음과 같다.

```text
d = n · p + c
```

직교 투영점은 다음과 같다.

```text
p' = p - n × d
```

표시 조건은 두 가지를 모두 만족해야 한다.

1. `abs(d) <= boreholeRadius` (절단 평면과 시추공 원통이 실제 교차)
2. 투영점의 단면 진행거리(chainage)가 시작점과 끝점 사이에 있음

현재 데이터는 수직 시추공이므로 한 시추공의 모든 지층 구간이 같은 수평 거리를 가진다. 향후 경사공 궤적을 지원할 때는 궤적 선분별 최소거리와 허용거리 경계 교점을 계산해 구간을 자른다.

## 6. 표현 규칙

- 근접 시추공을 단면으로 끌어오는 허용거리는 사용하지 않음
- 절단 평면과 시추공 원통이 실제 교차할 때만 단면 기둥 생성
- 원본 원통 대신 지층별 폭이 일정한 평면 리본 사용
- 지층 색상은 그대로 유지하고 시추공 전체 기둥의 외곽만 1px 선으로 표시
- 리본과 라벨은 단면 평면에 평행하게 배치
- 라벨은 앞면과 뒷면 메시를 분리하고 뒷면 U 좌표를 반전해 양쪽 시점 모두 정방향으로 표시
- 라벨 월드 크기는 카메라 거리에 비례해 조절하며 근거리 최소 배율은 0.18로 제한해 지층과 기둥을 과도하게 가리지 않음
- `depthTest=true`를 사용하고 단면 표면과의 z-fighting만 작은 polygon offset으로 방지
- 카메라를 회전해도 리본과 라벨은 단면 밖으로 돌출되지 않음
- 단면의 유한한 시작·끝 범위 밖에 있는 시추공은 표시하지 않음
- 현재 기본 절단 방향에서 `plane.distanceToPoint(position) >= 0`인 좌측 유지 반공간의 비관통 시추공은 시추공명만 표시
- 원본 시추공 기둥·외곽선·collar 마커는 단면 모드에서 숨겨 단면 밖으로 돌출되지 않도록 함
- 제거 반공간의 지표 표식도 숨겨 잘린 공간에 공중 부유하는 표식이 생기지 않도록 함
- 시추공 절단을 끄면 투영 그룹을 제거하고 모든 원본 시추공을 표시

## 7. 상태 및 수명주기

평면 위치, 방향, 시작·끝점, 시추공 형상 또는 수직 과장 배율이 바뀌면 단면 시추공 그룹을 다시 만든다. 기존 상태 호환을 위한 `boreholeProjectionDistanceM` 값은 단면 포함 판정에 사용하지 않으며 UI에서도 노출하지 않는다.

폐기 시 전용 geometry와 material을 dispose한다. 원본 라벨의 CanvasTexture는 공유하되 단면 그룹 정리 시 texture 자체는 dispose하지 않는다.

## 8. 검증 기준

1. 절단면 뒤쪽의 비관통 시추공은 시추공명만 보인다.
2. 절단 평면이 원통을 실제 통과한 시추공만 단면 기둥으로 표시된다.
3. 카메라를 임의 방향으로 회전해도 시추공이 단면 앞뒤로 돌출되지 않는다.
4. 단면 이동과 방향 반전 시 투영 결과가 즉시 갱신된다.
5. 단면 종료 시 일반 3D 시추공이 복원된다.
6. 시추공 선택/해제와 프로젝트별 저장 상태는 변경되지 않는다.
7. 지층·지표면·지하수 절단과 절단 캡은 기존 동작을 유지한다.
8. 단면 모드에서 비관통 원본 3D 기둥과 collar 마커는 보이지 않는다.
9. 단면을 종료하면 일반 3D의 원본 기둥·외곽선·collar 마커·시추공명이 모두 복원된다.
