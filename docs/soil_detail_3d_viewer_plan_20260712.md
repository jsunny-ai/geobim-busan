# 토사층 세분류 3D 지질 뷰어 구현계획

작성일: 2026-07-12

## 1. 표시 기준

- 토사층 세분류는 지층 평면/외부 표면에는 직접 해치로 표시하지 않는다.
- 지층 평면은 기존 대분류인 `토사`, `풍화암`, `연암`, `보통암`, `경암` 표시 체계를 유지한다.
- 시추공 기둥은 각 지층 행의 `soil_type`을 기준으로 토사 세분류 해치를 표시한다.
- 수직 단면은 토사 대분류 cap에 여러 세분류 해치를 합성해서 반복 표시하지 않는다.
- 토사층은 `매립토`, `퇴적토`, `풍화토` 등 세분류별 독립 솔리드로 분할하고, 해당 솔리드의 절단면에 해치를 표시한다.
- `토사 세분류 해치` 스위치는 시추공 컬럼과 수직 단면 세분류 밴드의 해치 표시 여부를 제어한다.
- `매립토`, `퇴적토`, `붕적토`, `풍화토` 등 세분류 항목은 개별 토글로 표시/숨김을 제어한다.

## 2. 구현 범위

### 2.1 UI

- `ViewerControls`의 토사층 세분류 범례를 개별 토글 UI로 제공한다.
- 각 세분류 토글은 현재 상태를 `켬`/`끔`으로 표시한다.
- 기존 `토사 세분류 해치` 스위치는 유지하되, 세분류 해치 레이어 표시 여부를 담당한다.

### 2.2 3D 모델

- 암반 계열 지층 메쉬는 기존 대분류 기준 보간/메쉬 생성 구조를 유지한다.
- 토사 대분류 솔리드가 세분류 솔리드와 중복 표시되지 않도록, 세분류 솔리드가 생성되면 기본 토사 솔리드는 숨긴다.
- 토사 세분류 솔리드는 각 시추공의 `soil_type`별 두께를 보간하고, 전체 토사 두께 안에서 비율 배분해 생성한다.
- 토사 세분류 솔리드의 수평 상·하면에는 해치 텍스처를 적용하지 않는다.
- 토사 세분류 솔리드의 수직면과 수직 단면 cap에는 CAD/DWG 스타일의 투명 선 해치만 적용한다.
- 해치 텍스처에는 색상 배경을 넣지 않고, 솔리드 색상과 해치 선을 분리한다.
- 시추공 컬럼 세그먼트에도 `soilDetail` 메타데이터를 부여하고 개별 세분류 토글을 반영한다.
- 세분류 개별 토글은 해당 세분류 솔리드, 해치, 시추공 컬럼 구간을 함께 표시/숨김 처리한다.

## 3. 검증 항목

- DB의 `soil_type` 세분류 값이 시추공 컬럼 해치에 반영된다.
- 토사 지층 단면에 여러 세분류 해치가 합성되어 반복 표시되지 않는다.
- 수직 단면에서 `매립토`, `퇴적토`, `퇴적모래` 등 실제 세분류 솔리드의 절단면이 CAD식 선 해치로 표시된다.
- 지층의 수평 상·하면에는 세분류 해치가 표시되지 않는다.
- 세분류 개별 토글을 끄면 해당 세분류 솔리드와 컬럼 구간이 함께 숨겨진다.
- `토사 세분류 해치`를 끄면 세분류 솔리드는 유지되고 해치 패턴만 제거된다.
- 지층 평면/외부 표면에는 세분류 해치가 표시되지 않는다.
