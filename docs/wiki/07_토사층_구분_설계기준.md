# 토사층 구분 설계 기준

> Graph parent: [[설계기준]]

> 작성일: 2026-07-13
> 적용 대상: GeoBIM DB 재추출, CSV/PDF 시추공 자료 적재, 3D 지질 뷰어 지층 솔리드, 수직 단면, 토글/범례 UI

## 1. 목적

이 문서는 토사층을 `토사` 하나로 병합하지 않고, 원본 주상도와 CSV에 포함된 토사 세분류를 GeoBIM 데이터와 3D 지질 뷰어에 유지하기 위한 설계 기준이다.

기존 06_토사세분류_해치표현기준이 해치와 패턴의 시각 표현 기준이라면, 이 문서는 그보다 앞단의 데이터 구분, 지층 솔리드 생성, 토글 동작, 검증 기준을 정의한다.

## 2. 기본 원칙

1. 원본 지층명은 항상 보존한다.
2. `매립토`, `퇴적토`, `붕적토`, `풍화토`, `퇴적점토`, `퇴적모래`, `퇴적자갈` 등은 모두 `토사`로 덮어쓰지 않는다.
3. 지층 모델의 대분류 계산에는 `strata_group = soil`을 사용하되, 화면 표시와 토글에는 `soil_type` 세분류를 사용한다.
4. 3D 뷰어에서는 해치만 따로 움직이지 않고, 세분류 지층 솔리드와 해치가 같은 토글 상태를 따른다.
5. 해치는 지층면의 수평 상부면보다 수직 단면과 절단면에서 판독되도록 한다.
6. 원본 PDF, CAD, CSV에 별도 범례가 있으면 GeoBIM 기본 매핑보다 원본 범례를 우선한다.

## 3. 기준 자료 요약

토사층 구분은 하나의 국제 해치 표준으로 결정되는 항목이 아니라, 지반공학 분류 체계와 프로젝트 원본 범례를 함께 적용해야 한다.

| 기준 | 핵심 내용 | GeoBIM 적용 |
|---|---|---|
| ASTM D2487 | 공학 목적의 토질 분류는 입도, 액성한계, 소성지수 등 실내시험 결과를 기반으로 한다. | 정밀 분류 또는 시험값이 있는 경우 `soil_type` 보정 근거로 사용 |
| ASTM D2488 | 현장·실내에서 시각 및 수작업으로 토사를 설명하고 식별하는 절차를 다룬다. 정밀 분류가 필요하면 D2487을 사용한다. | 시추주상도 원문 지층명, 육안 분류, 현장 설명을 보존하는 근거 |
| ISO 14688-1 | 공학 목적의 토질 식별과 설명 규칙을 정의하고, 프로젝트 특성에 따라 더 상세한 세분류가 필요할 수 있음을 인정한다. | GeoBIM 프로젝트별 세분류 확장과 원문 보존 원칙의 근거 |
| USDA NRCS soil texture | 모래, 실트, 점토 비율로 texture class를 계산한다. | 점토/실트/모래 계열 보조 분류와 보고서 설명에 참고 |

## 4. GeoBIM 토사층 분류 체계

GeoBIM에서는 지층을 두 단계로 다룬다.

```text
대분류 strata_group:
soil | weathered_rock | soft_rock | normal_rock | hard_rock | unknown

토사 세분류 soil_type:
매립토 | 붕적토 | 퇴적토 | 충적토 | 풍화토 |
퇴적점토 | 퇴적모래 | 퇴적자갈 |
점토 | 실트 | 모래 | 자갈 | 유기질토 | 혼합토 | 토사
```

`soil_type`은 `strata_group = soil` 안에서만 사용한다. `풍화토`는 토사 세분류이고, `풍화암`은 암반 대분류이므로 서로 병합하지 않는다.

## 5. 원문 지층명 매핑 기준

| 원문 예시 | 표준 soil_type | strata_group | 우선 근거 | 비고 |
|---|---|---|---|---|
| 매립토, 매립층, 성토, 되메움 | 매립토 | soil | 성인/시공 이력 | 인공 지층 성격 보존 |
| 붕적토, 붕적층 | 붕적토 | soil | 성인 | 사면/중력성 퇴적 성격 보존 |
| 퇴적토, 퇴적층 | 퇴적토 | soil | 성인/층서 | 세부 입도 정보가 없으면 퇴적토 유지 |
| 충적토, 충적층, 하상퇴적 | 충적토 | soil | 성인/수성 퇴적 | 원문에 충적이 있으면 퇴적토보다 우선 |
| 풍화토, 잔류토 | 풍화토 | soil | 풍화 잔류 토사 | 풍화암으로 변경하지 않음 |
| 퇴적점토, 점토질 퇴적토 | 퇴적점토 | soil | 성인 + 재료 | 퇴적 성격과 점토 성격 동시 보존 |
| 퇴적모래, 사질 퇴적토 | 퇴적모래 | soil | 성인 + 재료 | 퇴적 성격과 모래 성격 동시 보존 |
| 퇴적자갈, 자갈질 퇴적토 | 퇴적자갈 | soil | 성인 + 재료 | 퇴적 성격과 자갈 성격 동시 보존 |
| 점토, 점성토, CL, CH | 점토 | soil | 재료/USCS | 시험값이 있으면 USCS 기호 병기 |
| 실트, ML, MH | 실트 | soil | 재료/USCS | 점토와 구분 |
| 모래, 사질토, SW, SP, SM, SC | 모래 | soil | 재료/USCS | 혼합토는 주성분 규칙 적용 |
| 자갈, 역질토, GW, GP, GM, GC | 자갈 | soil | 재료/USCS | 큰 입자 지층 |
| 유기질토, 이탄, 부식토, OL, OH, PT | 유기질토 | soil | 재료/USCS | 별도 주의 표기 |
| 토사, 표토, 토층 | 토사 | soil | 원문 일반명 | 세분류 불명일 때만 사용 |

## 6. 우선순위 규칙

같은 지층명에 성인, 층서, 재료명이 함께 들어오면 다음 순서로 판단한다.

1. 원본 범례 또는 프로젝트별 매핑 테이블
2. 성인/층서명: 매립토, 붕적토, 충적토, 퇴적토, 풍화토
3. 복합명: 퇴적점토, 퇴적모래, 퇴적자갈
4. 재료명: 점토, 실트, 모래, 자갈, 유기질토
5. 일반명: 토사
6. 판독 불가: 미분류 또는 원문 유지 후 검수 대상으로 표시

이 순서는 3D 뷰어에서 토사층이 모두 같은 `토사` 솔리드로 합쳐지는 문제를 방지하기 위한 최소 기준이다.

## 7. DB 및 API 저장 기준

재추출 또는 CSV 업로드 후 지층 레코드는 다음 정보를 보존해야 한다.

```text
strata_name_raw       = 원본 PDF/CSV/OCR의 원문 지층명
strata_group          = soil | weathered_rock | soft_rock | normal_rock | hard_rock | unknown
soil_type             = 표준화된 토사 세분류명
soil_type_source      = original_legend | raw_name | uscs | visual_manual | inferred | user_override
soil_type_confidence  = high | medium | low
pattern_key           = 프론트엔드 또는 프로젝트 범례에서 계산 가능한 해치 키
```

`soil_type`이 비어 있는 토사층은 뷰어에서 `토사`로 표시할 수 있지만, DB 재추출 단계에서는 원문명이 소실되지 않아야 한다.

## 8. 3D 지질 뷰어 적용 기준

3D 뷰어는 토사 세분류를 해치만으로 표현하지 않고, 지층 자체를 분리된 표시 단위로 다룬다.

1. `soil_type`별로 `soil_detail:<세분류명>` 표시 레이어를 만든다.
2. 세분류 정보가 있는 프로젝트에서는 기본 `soil` 솔리드가 세분류 솔리드를 가리지 않도록 한다.
3. 토글은 `토사`, `풍화암`, `연암`과 같은 대분류 토글과 동일한 방식으로 동작한다.
4. `매립토`, `퇴적토`, `풍화토` 등 세분류 토글은 해당 세분류 솔리드, 단면 cap, 시추공 컬럼 세그먼트를 함께 제어한다.
5. 해치 표시 토글은 선 패턴의 표시 여부만 제어하고, 지층 솔리드의 표시 여부는 세분류 토글이 제어한다.
6. 지층 평면 상부면에는 해치를 기본 표시하지 않는다.
7. 수직 단면, 절단면 cap, 시추공 주상도에서 CAD 도면에 가까운 단색 선 해치를 표시한다.

## 9. 부산역 프로젝트 검증 기준

부산역 프로젝트처럼 CSV 원본에서 토사 세분류가 이미 존재하는 경우 다음을 확인한다.

1. 신규 시추공 134개가 기존 시추공으로 잘못 이동되지 않았는가.
2. 각 시추공 지층에서 `soil_type`이 `토사` 하나로 덮어써지지 않았는가.
3. `매립토`, `붕적토`, `퇴적토`, `퇴적점토`, `퇴적모래`, `퇴적자갈`, `풍화토`가 각각 별도 토글로 제어되는가.
4. 단면에서 세분류 솔리드와 해치가 같은 범위로 움직이는가.
5. 해치가 수평면에 과도하게 보이지 않고, 수직 단면에서 판독되는가.
6. 실측 수위 마커와 지하수위 포화 영역이 토사 세분류 판독을 방해하지 않는가.

## 10. 검수 체크리스트

| 단계 | 확인 항목 |
|---|---|
| 원본 적재 | CSV/PDF 원문 지층명이 `strata_name_raw`에 남아 있는가 |
| 표준화 | 토사 세분류가 `soil_type`으로 분리되었는가 |
| 대분류 | 모든 토사 세분류의 `strata_group`이 `soil`인가 |
| 모델 생성 | 세분류별 지층 솔리드가 생성되는가 |
| 토글 | 세분류 토글이 솔리드와 해치를 함께 제어하는가 |
| 단면 | 수직 단면 cap에 해치가 표시되는가 |
| 평면 | 지층 평면 상부면에는 해치가 기본 표시되지 않는가 |
| 범례 | 범례에 색상, 해치, 세분류명이 함께 표시되는가 |
| 보고 | 원본 범례와 GeoBIM 기본 기준 중 어떤 근거를 썼는지 남는가 |
