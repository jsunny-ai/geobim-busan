# Leapfrog Geo 및 지층 모델링의 미시추 최하부(경암층) 처리 기법 조사 보고서

시추공이 최하부 지층(기반암/경암)에 도달하지 못하고 중간에 종료(EOH: End of Hole)되었을 때, 최하단 경계와 미분류 구역을 어떻게 처리하는가에 대한 지질학적 학술 자료 및 상용 소프트웨어들의 기법 조사 결과입니다.

---

## 1. 리프로그(Leapfrog Geo)에서의 처리 기법

Leapfrog Geo에서는 시추공이 경암(Bedrock)에 도달하기 전에 조기 종료되어 최하부 데이터가 없는 상태에서 보간 엔진이 임의로 가상의 경암층을 생성(Smearing)하는 것을 방지하기 위해 다음 세 가지 핵심 기법을 적용합니다.

### ① 해석 테이블(Interpretation Table)을 통한 Lithology 보정
* **동작 원리**: Leapfrog 내부의 **Borehole Correlation** 도구를 활용해 2D 단면상에서 시추공들을 상호 연결하고, 실질적으로 경암에 도달하지 못한 EOH(End of Hole) 구간 아래에 가상의 'Unreached(미도달)' 또는 'Unsampled(미조사)' 구간을 인위적으로 병합하거나 수정해 넣습니다.
* **효과**: 보간 엔진이 빈 공간을 임의로 '경암'으로 해석하는 대신 '미조사(Null/Unknown)' 영역으로 인지하게 만듭니다.

### ② 맞춤형 모델 하부 경계(Custom Model Base) 설정
* **동작 원리**: 모델의 바닥을 단순 직사각형 상자 바닥으로 설정하지 않고, **New Base** 설정을 통해 지질 모델의 한계 곡면을 수립합니다.
  * **EOH Line**: 모든 시추공의 최하단 깊이를 연결한 포인트를 곡면화하여 모델의 Base로 정의합니다.
  * **물리탐사 데이터 융합**: 지전기 탐사(Resistivity)나 탄성파 탐사(Seismic) 등으로 추정한 기반암 심도면(TIN/Grid)을 모델의 Base Surface로 지정합니다.
* **효과**: 정의된 Base Surface 이하의 미시추 구역은 모델링 연산 대상에서 아예 제외(Inactive Cells)되거나 표시되지 않으므로, 데이터가 없는 영역에 거짓 지층이 모델링되는 것을 근본적으로 차단합니다.

### ③ Fix Errors 도구와 Omit/Replace 옵션
* **동작 원리**: 시추 테이블 끝단에 남겨진 빈 값(Null)이나 불일치 구간에 대해 Leapfrog는 데이터 경고(Red X)를 띄웁니다. 이때 사용자는 **Omit(제외)**하여 해당 깊이의 RBF 연산 참여를 배제하거나, **Replace(대체)** 기능을 사용해 `Unknown` 카테고리나 특정 불확실성 코드(예: -99)로 치환합니다.

## 2. 기타 상용 지층 가시화 프로그램의 기법

### ① RockWorks (지질공학 표준 소프트웨어)
* **Zero-Thickness & Isopach Clipping**:
  * 특정 구역에 기반암이 없거나 시추되지 않은 경우, 해당 좌표에서는 해당 지층의 두께를 `0`으로 처리하거나 '시추 한계선(EOH Filter)'을 적용해 렌더링을 완전히 생략(투명 처리)합니다.
  * 이를 통해 모델러가 탐사하지 않은 깊이를 특정 지층(경암)으로 착각하지 않도록 방지합니다.

### ② Petrel (Schlumberger 사의 오일/가스 지층 솔루션)
* **Active/Inactive Grid Cells 설정**:
  * 시추 데이터가 존재하는 3차원 바운딩 박스 내부에서도, 신뢰할 수 없는 심도 이하의 셀들은 `Inactive` 상태로 전환하여 격자 생성 시 투명하거나 빈 공간으로 마스킹합니다.

## 3. 학술 논문 및 지질공학 연구 동향

### ① 기반암 심도 예측 보간 기법 (*Lemon & Jones 2003*)
* 시추가 얕게 이루어진 구간에서 하부 경계를 단순 수학적 보간(RBF/TPS)에 의존하면 심각하게 왜곡된 가상의 지층(Ghost Layer)이 융기하거나 가라앉습니다.
* 논문에서는 이를 방지하기 위해 탄성파 탐사 결과 등의 보조 데이터를 **소프트 컨스트레인트(Soft Constraints, 약한 구속조건)**로 설정하여 보간면이 지질학적으로 타당한 하한선 내에서만 움직이도록 제한해야 함을 지적합니다.

### ② 불확실도 시각화와 미분류 복셀 (*Wu et al. 2003*, *Lixin et al. 2011*)
* 지하지반 모델링에서 정보가 결여된(Data-absent) 최하단 공간을 특정 암종으로 확정하여 가시화하는 것은 지반 조사 설계자에게 심각한 오판을 유발합니다.
* 따라서 최하부 데이터 결여 구간은 **미분류(UNKNOWN) 복셀**로 렌더링하거나, 정보의 불확실도를 엔트로피(Entropy Map) 형태로 색상 코딩하여 지질조사 보고서에 수록해야 한다고 명시하고 있습니다.

## 4. 현재 우리 프로젝트를 위한 해결 기술 제안

현재 우리 프로젝트 코드의 `classifyByRBF` 최하단 로직은 아래와 같이 무조건적인 경암(hard_rock) 반환 형태로 설계되어 있습니다.
```typescript
// rbfSurface.ts
if (elevation >= bndElev[0]) return 1  // soil
if (elevation >= bndElev[1]) return 2  // weathered_rock
if (elevation >= bndElev[2]) return 3  // soft_rock
return 4                                // ⚠️ 데이터 없음에도 무조건 경암(hard_rock)으로 채움
```

이 기하학적 왜곡을 해소하기 위한 두 가지 합리적인 엔지니어링 방안을 제안합니다.

### [대안 ①] 미분류(UNKNOWN)층 적용 (학술 및 논문 권장 표준)
* **설명**: 시추 데이터 전체 또는 특정 시추공에서 `hard_rock` 데이터가 전혀 취득되지 않은 경우, $bndElev[2]$(연암 하부 경계) 이하의 격자는 **`4(hard_rock)` 대신 `5(UNKNOWN/미분류)`**로 판정합니다.
* **장점**: 화면상에 지표 아래 깊은 미시추 구역이 회색의 '미분류층'으로 채워져, 데이터가 없어 확인되지 않은 영역임을 명확히 시각적으로 전달합니다.

### [대안 ②] 최하부 실측 지층의 연장 (현업 및 Leapfrog 권장 표준)
* **설명**: 시추공이 실제로 끝난 가장 깊은 지층(예: 연암)이 지반의 바닥까지 이어진 것으로 간주합니다.
* **장점**: 시추 데이터 내에 경암이 없는 프로젝트라면 모델 최하단이 경암층 대신 **실제 실측된 연암(soft_rock)층**으로 두껍게 덮이게 되어, 존재하지 않는 경암층이 렌더링되는 문제를 해결합니다.
